Facebook Phishing Techniques Analysis

Alternatif untuk About:Blank pada Phishing

Dalam penetration testing Facebook, berikut adalah beberapa teknik alternatif yang dapat digunakan selain about:blank:

Data URL
JavaScript Redirect
IFrame Injection
Domain Obfuscation

Teknik Data URL

Menggunakan data URL untuk membuat halaman yang terlihat legitimate:

// Contoh penggunaan data URL var newWindow = window.open('data:text/html;base64,PHNjcmlwdD5hbGVydCgiRGF0YSBVUkwgVGVjaG5pcXVlIik7PC9zY3JpcHQ+', '_blank');

JavaScript Redirect

Redirect setelah halaman dimuat untuk menghindari kecurigaan:

// Redirect setelah timer setTimeout(function() { window.location.href = "https://facebook.com"; }, 3000); // Atau redirect setelah interaksi pengguna document.getElementById('legitimateButton').addEventListener('click', function() { window.location.href = "https://facebook.com"; });

IFrame Injection

Menyisipkan konten phishing melalui iframe yang tidak terlihat:

// Menyembunyikan iframe dengan konten phishing var iframe = document.createElement('iframe'); iframe.style.display = 'none'; iframe.src = 'https://legitimate-site.com/phishing-page'; document.body.appendChild(iframe); // Atau iframe transparan yang menutupi seluruh halaman iframe.style.position = 'absolute'; iframe.style.top = '0'; iframe.style.left = '0'; iframe.style.width = '100%'; iframe.style.height = '100%'; iframe.style.opacity = '0';

Domain Obfuscation

Menggunakan teknik pengaburan domain untuk menipu pengguna:

// Domain dengan karakter unicode yang mirip (homograph attack) var domain = 'https://facebооk.com'; // Perhatikan 'о' adalah karakter cyrillic // Subdomain yang menipu var url = 'https://facebook.login.example.com'; // Menggunakan HTTPS untuk terlihat legitimate var secureUrl = 'https://secure-facebook-login.com';
Peringatan: Teknik-teknik ini disajikan untuk tujuan edukasi dan penetration testing yang sah saja. Penggunaan untuk aktivitas ilegal adalah tindakan kriminal dan dapat dikenakan konsekuensi hukum.

Rekomendasi Keamanan untuk Facebook

Berdasarkan analisis teknik phishing, berikut rekomendasi untuk meningkatkan keamanan:

1. Implementasi Content Security Policy (CSP)

Gunakan CSP header untuk mencegah eksekusi script berbahaya dan inline script.

2. X-Frame-Options Header

Atur X-Frame-Options ke "DENY" atau "SAMEORIGIN" untuk mencegah clickjacking.

3. Validasi Domain dan SSL

Lakukan validasi ketat terhadap domain dan sertifikat SSL untuk mencegah spoofing.

4. Edukasi Pengguna

Edukasi pengguna untuk selalu memeriksa URL dan tidak memasukkan kredensial di halaman mencurigakan.

5. Multi-Factor Authentication (MFA)

Terapkan MFA untuk melindungi akun bahkan jika kredensial dicuri.
